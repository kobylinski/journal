# reminder

A known issue you deliberately park - not a task on a queue, a note you can return to. Default
`status: confirmed`; open vs closed is tracked by `resolves`, not by status.

Worth capturing - include what applies, and anything else that matters:

- what the issue is
- why now is not the time
- what it risks if left

Without the reasoning it is a bare task, which belongs in the tracker, not the journal. When it
is dealt with, the closing entry (a `decision`, `tombstone`, or `summary`) carries `resolves`
back here.
