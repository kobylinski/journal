# reminder

Something set aside to return to - not a task on a queue, a note you can come back to. Two shapes
share this kind: an **open issue** parked for later, and a **piece of code worth preserving** as
the tree around it is torn out. Default `status: confirmed`; for a parked issue, open vs closed is
tracked by `resolves`, not by status.

Worth capturing - include what applies, and anything else that matters:

- a parked issue: what it is, why now is not the time, what it risks if left
- a preserved piece: why it was kept - a reusable pattern, code you mean to return to, a warning,
  or an exemplar worth remembering - quoted narrowly, with where it lived (see Compose, step 4)

Without the reasoning it is a bare task or a bare snippet, which belongs in the tracker or in git,
not the journal. When a parked issue is dealt with, the closing entry (a `decision`, `tombstone`,
or `summary`) carries `resolves` back here.
